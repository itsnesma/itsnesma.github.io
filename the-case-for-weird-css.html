The case for weird CSS

In defense of doing strange things with the browser instead of asking it to be polite.

CSS has a reputation as the language you tolerate. The thing you fight with to get the button aligned. The reason your Friday afternoon disappeared. I've heard "I hate CSS" from people who would never say "I hate Python" — even though both languages are about equally weird, just in different directions.

I think this reputation is unfair, and I think it's because most of us only ever use CSS for the boring eighty percent. Layout, spacing, type. Important work, but not the part that makes CSS fun. The fun part is everything else.

What weird CSS looks like

A few things I've made with CSS that I had no business making with CSS:

  • A rotating fan blade that uses an SVG goo filter to blur into a single shape.
  • Eyes that blink on a three-second timer using only @keyframes.
  • A character's mouth made entirely of repeating linear gradients (no images, no fonts, just math).

None of these are practical. All of them taught me something I now use in practical work. The eye-blink animation timing? That's the same easing curve I now use for menu reveals. The gradient mouth? That's the foundation of every textured background I make. Weird CSS is research and development for normal CSS.

Why it matters

Every tool gets richer the more weird things you've tried to do with it.

Frameworks have abstracted away most of the strange corners of CSS. That's mostly good — most of us shouldn't need to know how the cascade works to ship a button. But abstraction has a cost: the people who understand the weird corners are the people who can fix things when the abstractions leak. And they always leak.

So here's my pitch: every few months, build something useless. A little CSS-only character. A button that does too much. A loading animation that didn't need to exist. The web is more interesting when its builders remember that it can be.
